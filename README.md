## About Me

I'm a full-stack engineer who works primarily in **Go** and the **TypeScript/JavaScript** ecosystem, with a strong pull toward systems-level engineering — how schedulers, orchestrators, and concurrent runtimes actually work under the hood, not just how to call their APIs.

- 🔭 Currently building **goK8s**, a from-scratch reimplementation of core Kubernetes internals (API server, scheduler, kubelet) in Go
- 🧠 Deep interest in **Go's concurrency model** — goroutines, channels, and the GMP scheduler
- ☸️ Comfortable across the container/orchestration stack: **Docker, Kubernetes, OpenShift**
- 🌱 Currently sharpening system design fundamentals — data replication, contention handling, real-time delivery patterns, and scaling strategies
- 🐧 Daily driver: **Arch Linux**, **Neovim**
- 🎯 Actively open to **remote software engineering roles**

## Featured Projects

### ☸️ [goK8s](https://github.com/tekluabayneh/gok8s)
A ground-up reimplementation of core Kubernetes internals in Go — API server, etcd-backed object storage, watch mechanism, scheduler, and kubelet — built to understand control-loop systems at the source-code level, not just the CLI.

`Go` · `gRPC` · `Distributed Systems` · `Concurrency`

### 🛡️ [Edge-Aware Cybersecurity Orchestrator](https://github.com/tekluabayneh/Edge-Aware-Cybersecurity-Orchestrator)
A distributed threat-detection system for edge/IoT devices: a Go orchestrator, lightweight Go edge agents, and a Python ML-driven anomaly analyzer, coordinated over secure channels with a React dashboard for live visualization.

`Go` · `Python` · `TypeScript/React` · `Docker` · `gRPC`

### ✍️ [Collaborative Code Editor](https://github.com/tekluabayneh/colaborative_code_editor)
A real-time, multi-user code editor with a TypeScript backend and React frontend, deployed to a Kubernetes cluster (OpenShift/Red Hat Developer Sandbox) with a full CI pipeline and Cypress end-to-end test coverage.

`TypeScript` · `React` · `Kubernetes` · `Cypress`
